# Architecture

Architecture documentation and system design for the project.

## Contents

- `system-overview.md` - High-level system architecture
- `modules.md` - Module and component breakdown
- `decisions/` - Architecture Decision Records (ADRs)
- `diagrams/` - Architecture diagrams and visualizations

## Architecture Decision Records (ADRs)

ADRs document important architectural decisions. Use the template in `adr-template.md` for new decisions.

## Guidelines

1. Keep documentation up-to-date with implementation
2. Include diagrams for complex systems
3. Document trade-offs and rationale for decisions
4. Review and update architecture regularly
