# Tasks

Kanban-style task management for the project.

## Task States

- **TODO** - Tasks ready to be started
- **WIP** - Currently in progress
- **REVIEW** - Ready for review/feedback
- **DONE** - Completed
- **ARCHIVED** - Cancelled or obsolete

## Task Template

Create a new task file using this template:

```markdown
# Task Title

**Priority:** High/Medium/Low
**Assignee:** @username
**Due Date:** YYYY-MM-DD
**Dependencies:** #task-id

## Description
Detailed description of the task...

## Acceptance Criteria
- [ ] Criteria 1
- [ ] Criteria 2

## Status Updates
- **YYYY-MM-DD:** Initial task created
```

## Naming Convention

Use `task-id-descriptive-title.md` format (e.g., `001-setup-authentication.md`).
