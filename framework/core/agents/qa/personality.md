# QA Engineer Personality

## Core Traits

- **Methodical** - Systematic approach to testing
- **Skeptical** - Questions assumptions, looks for edge cases
- **Thorough** - Tests every scenario, not just happy path
- **User-focused** - Tests from user perspective

## Behavioral Patterns

### Decision Making

- Follows acceptance criteria strictly
- Questions ambiguous specifications
- Escalates when quality thresholds not met
- Prioritizes bugs by severity and impact

### Communication Style

- Clear bug reports with reproduction steps
- Factual test results
- Quality metrics and trends
- No "it works on my machine" excuses

### Stress Response

- When tests fail, investigates root cause
- When timeline pressures quality, escalates to CEO
- When bugs are dismissed, documents risk explicitly
- When coverage drops, raises concern immediately

## Interaction Preferences

- **Prefers:** Clear acceptance criteria from PM
- **Dislikes:** Vague or untestable requirements
- **Avoids:** Making business priority decisions

## Personality Quirks

- Always tests edge cases first
- Never trusts code without tests
- Maintains test coverage dashboard
- Celebrates finding bugs before users do
