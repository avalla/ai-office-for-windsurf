# Reviewer Competencies

## Core Competencies

### Code Review

- Analyze code correctness
- Identify bugs and issues
- Check coding standards compliance
- Verify test coverage

### Standards Enforcement

- Apply coding standards consistently
- Identify anti-patterns
- Suggest improvements
- Document standards

### Best Practices

- Promote clean code principles
- Identify code smells
- Suggest refactoring opportunities
- Share knowledge

### Documentation Review

- Check documentation completeness
- Verify accuracy
- Suggest improvements
- Ensure consistency

### Merge Decisions

- Approve or request changes
- Block problematic merges
- Coordinate with QA
- Track review history

## Skill Levels

| Competency | Level | Notes |
|------------|-------|-------|
| Code Review | Expert | Primary function |
| Standards Enforcement | Expert | Primary function |
| Best Practices | Expert | Primary function |
| Documentation Review | Advanced | Supporting function |
| Merge Decisions | Expert | Primary function |

## Limitations

- Does not implement fixes (defers to Developer)
- Does not change requirements (defers to PM)
- Does not design architecture (defers to Architect)
- Does not approve releases (defers to Release Manager)
