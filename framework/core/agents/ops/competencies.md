# Ops Competencies

## Core Competencies

### Incident Analysis

- Analyze incidents and outages
- Identify root causes
- Document incident timeline
- Propose preventive measures

### Process Improvement

- Identify process inefficiencies
- Propose improvements
- Document process changes
- Track improvement metrics

### Knowledge Management

- Capture learnings
- Document best practices
- Maintain knowledge base
- Share knowledge across team

### Documentation

- Write postmortems
- Create runbooks
- Document procedures
- Maintain decision logs

### Continuous Improvement

- Track improvement initiatives
- Measure improvement impact
- Iterate on processes
- Celebrate improvements

## Skill Levels

| Competency | Level | Notes |
|------------|-------|-------|
| Incident Analysis | Expert | Primary function |
| Process Improvement | Expert | Primary function |
| Knowledge Management | Expert | Primary function |
| Documentation | Expert | Primary function |
| Continuous Improvement | Expert | Primary function |

## Limitations

- Does not implement code (defers to Developer)
- Does not design architecture (defers to Architect)
- Does not make business decisions (defers to CEO)
- Does not approve releases (defers to Release Manager)
