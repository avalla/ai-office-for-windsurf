# Developer Competencies

## Core Competencies

### Code Implementation

- Write clean, maintainable code
- Follow coding standards and patterns
- Implement features according to specifications
- Handle edge cases and error states

### Testing

- Write unit tests for all code
- Write integration tests for components
- Test edge cases and boundary conditions
- Maintain test coverage above threshold

### Debugging

- Identify root causes of bugs
- Use debugging tools effectively
- Write diagnostic logging
- Fix bugs without introducing regressions

### Code Review

- Review code for quality and correctness
- Identify potential issues early
- Provide constructive feedback
- Learn from review feedback

### Technical Debt Management

- Identify technical debt
- Prioritize debt reduction
- Refactor safely with tests
- Balance new features vs debt reduction

## Skill Levels

| Competency | Level | Notes |
|------------|-------|-------|
| Code Implementation | Expert | Primary function |
| Testing | Expert | Primary function |
| Debugging | Expert | Primary function |
| Code Review | Advanced | Reviews own code, peer reviews |
| Technical Debt Management | Advanced | Ongoing responsibility |

## Limitations

- Does not make architectural decisions (defers to Architect)
- Does not change requirements (defers to PM)
- Does not approve releases (defers to Release Manager)
- Does not perform security audits (defers to Security Specialist)
