# Game Developer Triggers

## Primary Triggers

### Slash Commands

| Command | Action |
|---------|--------|
| `/28_game_create` | Game development workflow |

### Workflow Events

| Event | Action |
|-------|--------|
| Game design approved | Implement game systems |
| Assets ready | Integrate into game |
| Playtest feedback | Iterate on mechanics |
| Performance issues | Optimize game |

## Secondary Triggers

### Context-Based

- Designer provides game UI
- Audio Creator provides game audio
- Image Creator provides game assets
- PM requests game feature

### Escalation-Based

- Critical gameplay bugs
- Performance bottlenecks
- Platform certification issues
- Player feedback issues

## Activation Conditions

### Required For

- **Game Projects** - All game development
- **Interactive Experiences** - Game-like applications
- **AR/VR Projects** - Immersive development

### Optional For

- PRD review (game feasibility)
- Architecture review (game systems)
- Release review (game performance)

## Trigger Examples

### Example 1: Game Project

```
CEO: Approves game design document
Game Developer: Implements core mechanics, systems, and gameplay
```

### Example 2: Asset Integration

```
Audio Creator: "Game audio assets ready"
Game Developer: Integrates audio with interactive music system
```

### Example 3: Playtest Feedback

```
QA: "Players struggle with level 3 difficulty"
Game Developer: Adjusts difficulty curve, adds tutorial hints
```
