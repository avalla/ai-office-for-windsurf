# UX Researcher Skills

## Skills Used

| Skill | File | Usage |
|-------|------|-------|
| `review-document-multisector` | `review-document-multisector.md` | Research report review |
| `project-analysis` | `project-analysis.md` | Understand existing user flows |

## Skill Usage Patterns

### review-document-multisector

**When Used:**
- Research report validation
- Persona documentation review

**How Used:**
- Validate research completeness
- Check user perspective coverage
- Ensure actionable insights

### project-analysis

**When Used:**
- Understanding existing user flows
- Analyzing current UX patterns
- Identifying improvement opportunities

**How Used:**
- Map existing user journeys
- Identify pain points
- Document current state

## Skills NOT Used

UX Researcher does NOT use:
- `run-tests` - Testing is QA responsibility
- `generate-tests` - Testing is QA responsibility
- `refactor-safe` - Implementation is Developer responsibility
- `security-check` - Security is Security Specialist responsibility
- `generate-migration` - Database work is Developer responsibility

## Skill Invocation

UX Researcher focuses on research and analysis:

```markdown
1. Define research questions
2. Invoke project-analysis (if existing product)
3. Conduct research
4. Analyze and synthesize
5. Invoke review-document-multisector (report)
6. Present insights
```
