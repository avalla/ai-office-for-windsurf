# Tokenomics Strategist MCP Adapters

## Available Adapters

| Adapter | Server | Usage |
|---------|--------|-------|
| `fetch` | `fetch` | Research protocol patterns, market references, and mechanism examples |
| `sequential-thinking` | `sequential-thinking` | Evaluate multi-scenario incentive trade-offs |
| `supabase` | `supabase` | Validate metric schema ideas and KPI tracking structures |

## Adapter Usage Patterns

### fetch

When Used:
- Review comparable token mechanisms
- Gather ecosystem and market context
- Benchmark incentive design patterns

### sequential-thinking

When Used:
- Explore competing tokenomics options
- Stress-test assumptions
- Sequence phased rollout strategies

### supabase

When Used:
- Define telemetry requirements for KPI tracking
- Validate reporting data model assumptions

## Adapters NOT Used

Tokenomics Strategist does NOT use:
- `snyk` - Security scanning belongs to Security Specialist
- `playwright` - E2E UI testing belongs to QA
- `runcomfy` - Creative asset generation belongs to creative agents
- `revenuecat` - Billing execution belongs to product/revenue implementation owners

## Adapter Constraints

- No direct implementation ownership
- No production deployment authority
- Focused on mechanism strategy and measurable outcomes
