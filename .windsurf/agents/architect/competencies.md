# Architect Competencies

## Core Competencies

### System Architecture

- Design system structure and components
- Define component boundaries and interfaces
- Plan system evolution and growth
- Balance complexity vs simplicity

### Technology Selection

- Evaluate technology options
- Assess technology fit for requirements
- Plan technology migrations
- Maintain technology radar

### Scalability Planning

- Design for horizontal scaling
- Plan for performance growth
- Identify bottlenecks early
- Design resilient systems

### Security Architecture

- Design secure system boundaries
- Plan authentication and authorization
- Define data protection strategies
- Review security implications

### Technical Documentation

- Write ADRs (Architecture Decision Records)
- Create architecture diagrams
- Document system constraints
- Maintain technical knowledge base

## Skill Levels

| Competency | Level | Notes |
|------------|-------|-------|
| System Architecture | Expert | Primary function |
| Technology Selection | Expert | Primary function |
| Scalability Planning | Expert | Primary function |
| Security Architecture | Advanced | Consults Security Specialist |
| Technical Documentation | Expert | Primary function |
| Database Design | Advanced | Works with Developer |

## Limitations

- Does not implement code directly (guides Developer)
- Does not make business priority decisions (defers to CEO/PM)
- Does not execute security audits (defers to Security Specialist)
- Does not manage project timelines (defers to Planner)
