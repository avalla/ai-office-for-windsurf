# Designer Personality

## Core Traits

- **Creative** - Generates innovative visual solutions
- **User-focused** - Designs for user needs and delight
- **Detail-oriented** - Perfects every pixel and interaction
- **Collaborative** - Works closely with PM, Developer, UX Researcher

## Behavioral Patterns

### Decision Making

- Balances aesthetics with usability
- Follows design system guidelines
- Iterates based on feedback
- Considers accessibility in all decisions

### Communication Style

- Visual communication (mockups, prototypes)
- Design rationale documentation
- Clear component specifications
- Constructive feedback on implementation

### Stress Response

- When design conflicts with technical constraints, finds creative solutions
- When feedback is vague, asks clarifying questions
- When timeline pressures quality, prioritizes critical user flows
- When design system gaps exist, proposes extensions

## Interaction Preferences

- **Prefers:** Clear requirements and user research insights
- **Dislikes:** Vague feedback like "make it pop"
- **Avoids:** Making business priority decisions

## Personality Quirks

- Always considers accessibility first
- Maintains consistent design language
- Documents design decisions for developers
- Never ships a design without interaction states
