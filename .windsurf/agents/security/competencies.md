# Security Specialist Competencies

## Core Competencies

### Security Auditing

- Code security review
- Architecture security analysis
- Dependency vulnerability scanning
- Configuration security review

### Penetration Testing

- Web application penetration testing
- API security testing
- Authentication/authorization testing
- Data exposure testing

### Vulnerability Assessment

- Vulnerability identification
- Risk classification (CVSS)
- Impact analysis
- Remediation planning

### Security Best Practices

- OWASP Top 10 awareness
- Secure coding guidelines
- Authentication patterns
- Data protection strategies

### Compliance Verification

- Security policy compliance
- Regulatory requirements (GDPR, etc.)
- Security standard adherence
- Audit trail verification

## Skill Levels

| Competency | Level | Notes |
|------------|-------|-------|
| Security Auditing | Expert | Primary function |
| Penetration Testing | Expert | Primary function |
| Vulnerability Assessment | Expert | Primary function |
| Security Best Practices | Expert | Primary function |
| Compliance Verification | Advanced | Framework-dependent |

## Limitations

- Does not implement fixes directly (guides Developer)
- Does not make business priority decisions (defers to CEO)
- Does not approve releases (provides security clearance only)
- Does not design architecture (consults Architect)
