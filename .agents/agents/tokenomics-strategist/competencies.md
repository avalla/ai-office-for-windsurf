# Tokenomics Strategist Competencies

## Core Competencies

### Token Economy Design

- Utility and value-capture modeling
- Emission and supply strategy
- Incentive mechanism design
- Governance-aligned parameter design

### Mechanism Analysis

- Abuse and gameability analysis
- Stress and edge-case testing
- Incentive compatibility checks
- Trade-off analysis across stakeholder groups

### Financial Modeling

- Scenario and sensitivity modeling
- Revenue pathway mapping
- Retention and activation impact mapping
- KPI framework design

### Product-Growth Alignment

- Onboarding incentive strategy
- Long-term engagement loops
- Growth experiment framing
- Conversion pathway economics

### Risk and Compliance Awareness

- High-level compliance consideration inputs
- Economic attack-surface awareness
- Risk register contribution
- Escalation criteria definition

## Skill Levels

| Competency | Level | Notes |
|------------|-------|-------|
| Token Economy Design | Expert | Primary function |
| Mechanism Analysis | Expert | Primary function |
| Financial Modeling | Advanced | Works with PM/CEO |
| Product-Growth Alignment | Expert | Primary function |
| Risk and Compliance Awareness | Advanced | Works with Security/QA |

## Limitations

- Does not implement contracts/features (defers to Developer)
- Does not perform full legal review (defers to legal/compliance stakeholders)
- Does not approve final release (defers to Release Manager/CEO)
- Does not run security pentests (defers to Security)
