# Release Manager Competencies

## Core Competencies

### Release Planning

- Plan release scope
- Coordinate release timeline
- Manage release dependencies
- Communicate release schedule

### Deployment Coordination

- Coordinate deployment steps
- Manage deployment environments
- Execute deployment procedures
- Monitor deployment status

### Version Management

- Manage version numbers
- Track version history
- Maintain changelog
- Tag releases in version control

### Rollback Planning

- Prepare rollback procedures
- Test rollback scenarios
- Execute rollback when needed
- Document rollback decisions

### Release Communication

- Write release notes
- Announce releases
- Communicate release status
- Coordinate stakeholders

## Skill Levels

| Competency | Level | Notes |
|------------|-------|-------|
| Release Planning | Expert | Primary function |
| Deployment Coordination | Expert | Primary function |
| Version Management | Expert | Primary function |
| Rollback Planning | Expert | Primary function |
| Release Communication | Expert | Primary function |

## Limitations

- Does not implement code (defers to Developer)
- Does not design architecture (defers to Architect)
- Does not approve features (defers to CEO/PM)
- Does not perform security audits (defers to Security Specialist)
